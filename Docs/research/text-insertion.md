# Research: text-insertion

## Summary
Pour insérer du texte transcrit au point d'insertion dans n'importe quelle app macOS, trois méthodes existent et aucune ne fonctionne dans TOUS les cas — la stratégie gagnante est le presse-papiers + simulation Cmd+V via CGEvent, avec sauvegarde/restauration du presse-papiers marquée par un jeton de session. C'est exactement ce que fait VoiceInk (dossier VoiceInk/Paste/: CursorPaster.swift, ClipboardManager.swift, PasteMethod.swift). VoiceInk N'utilise PAS l'API Accessibility (AX) pour écrire — l'AX ne sert qu'à LIRE le texte sélectionné (SelectedTextService via la lib SelectedTextKit). Son pipeline de paste offre deux méthodes : (1) « Default » = poser Cmd+V par CGEvent (virtualKey 0x37 Command + 0x09 V, flags .maskCommand, tap .cghidEventTap), (2) « AppleScript » = keystroke "v" ou key code 9 (fallback pour dispositions non-QWERTY qui remappent sur Command). La restauration du presse-papiers est faite proprement : snapshot de TOUS les NSPasteboardItem (type+Data), écriture du texte avec un type custom « com.VoiceInk.PasteSession » = UUID + les types transient de nspasteboard.org, puis restauration APRÈS un délai (min 0,25 s) UNIQUEMENT si le presse-papiers appartient encore à la session (string == texte attendu ET session == UUID) — ce qui évite d'écraser ce que l'utilisateur a copié entre-temps et n'utilise donc pas changeCount mais un marqueur de contenu, plus robuste. La méthode (b) CGEventKeyboardSetUnicodeString a une limite RÉELLE de 20 unités UTF-16 par événement (tronque silencieusement au-delà, confirmé par enigo) et échoue silencieusement si un chunk commence par \t \r \n (contournement : préfixer un espace zéro-largeur ​) — utile en fallback « type-out » mais lente et sujette aux pertes de caractères sous charge. La méthode (c) AX (kAXSelectedTextAttribute sur l'élément focalisé) insère au caret sans toucher au presse-papiers mais échoue dans Electron/Chromium web/terminaux et a une limite ~2040 caractères — donc peu fiable comme voie principale. AUCUNE méthode ne marche dans un champ « secure input » (mot de passe) : le secure event input bloque les event taps ET les événements synthétiques ; il faut le détecter via IsSecureEventInputEnabled() (Carbon) et prévenir l'utilisateur. La permission requise est Accessibility (AXIsProcessTrusted / AXIsProcessTrustedWithOptions avec kAXTrustedCheckOptionPrompt, deep-link vers le volet Confidentialité, polling 1 s). Gotcha SPM sans Xcode : sans signature de code (au moins ad-hoc) et bundle .app stable, l'octroi TCC Accessibility saute à chaque rebuild.</summary>
<recommendations>
<recommendation>
<topic>Méthode principale d'insertion</topic>
<recommendation>Utiliser presse-papiers + Cmd+V simulé par CGEvent comme voie principale (comme VoiceInk). C'est la seule méthode fiable dans quasi toutes les apps : natives Cocoa, navigateurs (contenu web contenteditable), Electron (VS Code/Slack/Discord), terminaux. Créer les événements avec CGEventSource(stateID: .privateState), virtualKey 0x37 (Command) puis 0x09 (V) avec flags = .maskCommand, poster sur tap .cghidEventTap. Insertion instantanée quelle que soit la longueur du texte (contrairement à la frappe unicode).</recommendation>
<rationale>L'AX en écriture (kAXSelectedTextAttribute) est inconsistante : elle échoue sur Chromium/Electron web-content et la plupart des terminaux, et plafonne ~2040 caractères. La frappe unicode (CGEventKeyboardSetUnicodeString) est limitée à 20 UTF-16/événement, lente et perd des caractères sous charge. VoiceInk, produit commercial open source concurrent de superwhisper/Wispr Flow, a convergé vers Cmd+V comme défaut précisément pour ces raisons de couverture d'apps.</rationale>
</recommendation>
<recommendation>
<topic>Sauvegarde/restauration du presse-papiers</topic>
<recommendation>Snapshotter TOUS les NSPasteboardItem (chaque type + Data), écrire le texte avec un marqueur de session (type custom UUID) + les types transient de nspasteboard.org (org.nspasteboard.TransientType, AutoGeneratedType, source=bundleID), puis restaurer APRÈS un délai (>= 0,25 s) UNIQUEMENT si le presse-papiers appartient toujours à la session (string encore == texte ET marqueur session encore présent). Ne PAS restaurer immédiatement — laisser à l'app cible le temps de lire le presse-papiers.</rationale>
<rationale>Restaurer trop tôt fait paster l'ancien contenu ; restaurer inconditionnellement écrase ce que l'utilisateur a copié entre-temps. Le marqueur de contenu (plus fiable que changeCount seul) et les types transient font que les gestionnaires de presse-papiers (Paste, Maccy, Raycast) ignorent l'entrée éphémère. Rendre la restauration optionnelle via un réglage (VoiceInk: clé UserDefaults "restoreClipboardAfterPaste", délai "clipboardRestoreDelay").</rationale>
</recommendation>
<recommendation>
<topic>Fallbacks en cascade</topic>
<recommendation>Proposer un mode « type-out » optionnel en fallback : frappe caractère-par-caractère via CGEventKeyboardSetUnicodeString par chunks de 20 unités UTF-16, en préfixant un espace zéro-largeur ​ devant tout chunk commençant par \t/\r/\n. À réserver aux cas où l'utilisateur veut préserver le presse-papiers coûte que coûte ou aux champs qui refusent Cmd+V. Prévoir aussi le fallback AppleScript (keystroke "v" / key code 9) pour les dispositions clavier qui remappent sur Command (nom de source se terminant par ⌘).</recommendation>
<rationale>Le chunk de 20 et le contournement du ​ sont confirmés par la lib enigo (référence largement utilisée) : CGEventKeyboardSetUnicodeString tronque silencieusement au-delà de 20 et échoue silencieusement sur un chunk à espace/contrôle en tête. Le key code 9 contourne les dispositions « X – QWERTY ⌘ » où keystroke "v" résout la mauvaise touche.</rationale>
</recommendation>
<recommendation>
<topic>Détection secure input</topic>
<recommendation>Avant chaque insertion, appeler IsSecureEventInputEnabled() (import Carbon). Si true, ne PAS tenter le paste (il échouera silencieusement) : afficher un avertissement style « Un champ sécurisé (mot de passe) est actif — MoDict ne peut pas y insérer de texte ». Optionnellement, identifier l'app fautive via ioreg/CGSSession (kCGSSessionSecureInputPID) mais IsSecureEventInputEnabled() suffit pour l'UX d'avertissement.</rationale>
<rationale>Le secure event input bloque à la fois les event taps ET l'injection d'événements synthétiques ET l'AX — donc les 3 méthodes échouent. Le détecter évite un échec silencieux déroutant. Terminaux (iTerm2/Terminal option « Secure Keyboard Entry »), gestionnaires de mots de passe et champs password navigateurs l'activent.</rationale>
</recommendation>
<recommendation>
<topic>Permission Accessibility + UX</topic>
<recommendation>Vérifier AXIsProcessTrusted() ; pour demander, appeler AXIsProcessTrustedWithOptions([kAXTrustedCheckOptionPrompt.takeUnretainedValue() as String: true]) ET deep-linker vers Réglages via NSWorkspace.shared.open(URL(string:"x-apple.systempreferences:com.apple.preference.security?Privacy_Accessibility")), puis poller AXIsProcessTrusted() toutes les 1 s (max ~60 s) pour détecter l'octroi et avancer l'onboarding automatiquement.</recommendation>
<rationale>C'est le pattern exact de VoiceInk (OnboardingPermissionController.swift). Poster un Cmd+V synthétique nécessite la permission Accessibility ; sans elle, CGEvent.post échoue. Le prompt système seul est insuffisant (l'app doit être déjà lancée pour apparaître dans la liste), d'où le deep-link + polling.</rationale>
</recommendation>
<recommendation>
<topic>Gotcha build SPM sans Xcode (TCC)</topic>
<recommendation>Empaqueter MoDict en vrai bundle .app avec un bundle identifier stable et le SIGNER (au moins ad-hoc: codesign --force --deep --sign - MoDict.app, idéalement avec un cert Developer ID). Sinon, chaque rebuild change l'identité du binaire et macOS révoque l'octroi Accessibility déjà accordé, forçant l'utilisateur à re-cocher la case à chaque compilation.</recommendation>
<rationale>TCC lie l'autorisation Accessibility à l'identité de code (signature/chemin). Un binaire non signé reconstruit via `swift build` casse l'octroi silencieusement — problème classique des apps macOS buildées hors Xcode. Une signature ad-hoc stable + bundle .app fixe résout ce point pour le workflow SPM/Command Line Tools de MoDict.</rationale>
</recommendation>
</recommendations>
<code_notes>
APIS EXACTES ET CONSTANTES
- Key codes (Carbon kVK_*): Command = 0x37, V = 0x09, Return = 0x24, Shift = flags .maskShift.
- CGEvent: CGEventSource(stateID: .privateState) ; CGEvent(keyboardEventSource:virtualKey:keyDown:) ; event.flags = .maskCommand ; event.post(tap: .cghidEventTap).
- Frappe unicode: CGEvent.keyboardSetUnicodeString(stringLength:unicodeString:) — LIMITE 20 unités UTF-16/événement (tronque au-delà). Échoue silencieusement si le chunk commence par \t/\r/\n → préfixer \u{200B}.
- AX écriture (méthode c, non recommandée en principal): AXUIElementCreateSystemWide() → AXUIElementCopyAttributeValue(system, kAXFocusedUIElementAttribute, &focused) → AXUIElementSetAttributeValue(focused, kAXSelectedTextAttribute as CFString, text as CFString). ~2040 char limite ; échoue Electron/web/terminaux.
- Secure input: import Carbon ; IsSecureEventInputEnabled() -> Bool. (EnableSecureEventInput/DisableSecureEventInput existent aussi mais réservés à qui SAISIT des mots de passe.)
- Permission: AXIsProcessTrusted() ; AXIsProcessTrustedWithOptions([kAXTrustedCheckOptionPrompt.takeUnretainedValue() as String: true]) ; deep-link "x-apple.systempreferences:com.apple.preference.security?Privacy_Accessibility".
- Presse-papiers: NSPasteboard.general.clearContents() ; setString(_,forType:.string) ; pasteboardItems ; writeObjects([NSPasteboardItem]) ; types transient: NSPasteboard.PasteboardType("org.nspasteboard.TransientType"), "org.nspasteboard.AutoGeneratedType", "org.nspasteboard.source".

VALEURS DE TIMING VOICEINK (référence produit)
- prePasteDelay = 0.10 s (avant de poser Cmd+V).
- délai entre chaque key event = 0.01 s.
- minimumClipboardRestoreDelay = 0.25 s (configurable via UserDefaults "clipboardRestoreDelay").
- frappe unicode enigo: ~4 ms entre chunks.

CODE SWIFT RECOMMANDÉ (voie principale, autonome, macOS 14+) :

import AppKit
import Carbon            // IsSecureEventInputEnabled
import ApplicationServices

enum InsertResult { case inserted, secureInputBlocked, noAccessibilityPermission, failed }

@MainActor
final class TextInserter {
    static let shared = TextInserter()

    // marqueur de session pour une restauration sûre du presse-papiers
    private let sessionType = NSPasteboard.PasteboardType("com.modict.PasteSession")
    private let transientType = NSPasteboard.PasteboardType("org.nspasteboard.TransientType")
    private let autoGenType   = NSPasteboard.PasteboardType("org.nspasteboard.AutoGeneratedType")
    private let sourceType    = NSPasteboard.PasteboardType("org.nspasteboard.source")

    func insert(_ text: String, restoreClipboard: Bool = true) async -> InsertResult {
        guard !text.isEmpty else { return .inserted }
        // 1) champ sécurisé -> impossible, prévenir
        if IsSecureEventInputEnabled() { return .secureInputBlocked }
        // 2) permission Accessibility requise pour poster Cmd+V
        guard AXIsProcessTrusted() else { return .noAccessibilityPermission }

        let pb = NSPasteboard.general
        let snapshot: [[(NSPasteboard.PasteboardType, Data)]] = restoreClipboard
            ? (pb.pasteboardItems ?? []).map { item in
                item.types.compactMap { t in item.data(forType: t).map { (t, $0) } } }
            : []
        let sessionID = UUID().uuidString

        // 3) écrire le texte + marqueurs
        pb.clearContents()
        guard pb.setString(text, forType: .string) else { return .failed }
        if let bid = Bundle.main.bundleIdentifier { pb.setString(bid, forType: sourceType) }
        if restoreClipboard {
            pb.setData(Data(), forType: transientType)
            pb.setData(Data(), forType: autoGenType)
            pb.setString(sessionID, forType: sessionType)
        }

        try? await Task.sleep(nanoseconds: 100_000_000) // prePasteDelay 0.10s

        // 4) poser Cmd+V
        let src = CGEventSource(stateID: .privateState)
        guard let cmdDown = CGEvent(keyboardEventSource: src, virtualKey: 0x37, keyDown: true),
              let vDown   = CGEvent(keyboardEventSource: src, virtualKey: 0x09, keyDown: true),
              let vUp     = CGEvent(keyboardEventSource: src, virtualKey: 0x09, keyDown: false),
              let cmdUp   = CGEvent(keyboardEventSource: src, virtualKey: 0x37, keyDown: false)
        else { return .failed }
        cmdDown.flags = .maskCommand; vDown.flags = .maskCommand; vUp.flags = .maskCommand
        cmdDown.post(tap: .cghidEventTap)
        try? await Task.sleep(nanoseconds: 10_000_000)
        vDown.post(tap: .cghidEventTap)
        try? await Task.sleep(nanoseconds: 10_000_000)
        vUp.post(tap: .cghidEventTap)
        try? await Task.sleep(nanoseconds: 10_000_000)
        cmdUp.post(tap: .cghidEventTap)

        // 5) restauration différée, seulement si on possède encore la session
        if restoreClipboard {
            try? await Task.sleep(nanoseconds: 250_000_000) // >= 0.25s
            let stillOurs = pb.string(forType: .string) == text
                         && pb.string(forType: sessionType) == sessionID
            if stillOurs {
                pb.clearContents()
                if !snapshot.isEmpty {
                    pb.writeObjects(snapshot.map { itemSnap in
                        let it = NSPasteboardItem()
                        for (t, d) in itemSnap { it.setData(d, forType: t) }
                        return it
                    })
                }
            }
        }
        return .inserted
    }
}

FALLBACK FRAPPE UNICODE (mode type-out, sans presse-papiers) :

func typeUnicode(_ text: String) {
    let src = CGEventSource(stateID: .privateState)
    let scalars = Array(text.utf16)
    var i = 0
    while i < scalars.count {
        let end = min(i + 20, scalars.count)             // limite 20 UTF-16
        var chunk = Array(scalars[i..<end])
        // \t/\r/\n en tête -> CGEvent échoue en silence : préfixer ZWSP
        if let f = chunk.first, f == 0x09 || f == 0x0A || f == 0x0D {
            chunk.insert(0x200B, at: 0)
        }
        if let e = CGEvent(keyboardEventSource: src, virtualKey: 0, keyDown: true) {
            e.keyboardSetUnicodeString(stringLength: chunk.count, unicodeString: chunk)
            e.post(tap: .cghidEventTap)
        }
        i = end
        usleep(4000) // ~4ms entre chunks
    }
}

<pitfalls>
<pitfall>AUCUNE méthode n'insère dans un champ « secure input » (mot de passe). Le secure event input bloque event taps, événements synthétiques ET l'AX. Toujours tester IsSecureEventInputEnabled() en premier et avertir, sinon échec silencieux déroutant.</pitfall>
<pitfall>CGEventKeyboardSetUnicodeString tronque SILENCIEUSEMENT au-delà de 20 unités UTF-16 par événement (confirmé enigo). Il faut chunker par 20. De plus un chunk commençant par \t, \r ou \n échoue en silence — contournement: préfixer un espace zéro-largeur \u{200B}.</pitfall>
<pitfall>Restaurer le presse-papiers trop tôt fait coller l'ANCIEN contenu (course avec l'app cible). Restaurer inconditionnellement écrase ce que l'utilisateur a copié entre-temps. Solution: délai >= 0,25 s ET restaurer seulement si le presse-papiers appartient encore à la session (marqueur UUID + texte inchangé), pas seulement via changeCount.</pitfall>
<pitfall>L'AX en écriture (kAXSelectedTextAttribute / kAXValue) est peu fiable: échoue sur Chromium/Electron (contenu web contenteditable) et la plupart des terminaux, plafonne ~2040 caractères, et setting kAXValueAttribute remplace TOUT le champ au lieu d'insérer au caret. Ne pas en faire la voie principale.</pitfall>
<pitfall>Dispositions clavier qui remappent sur Command (nom de source finissant par ⌘, ex. « X – QWERTY ⌘ »): keystroke "v" résout la mauvaise touche. Utiliser key code 9 (physique V) via AppleScript, ou poster directement le virtualKey 0x09 par CGEvent (non affecté par le remap logique).</pitfall>
<pitfall>Build SPM sans Xcode: un binaire non signé reconstruit change d'identité de code → macOS révoque l'octroi TCC Accessibility à chaque rebuild. Empaqueter en .app avec bundle id stable + signature (au moins ad-hoc `codesign -s -`) pour que la permission persiste.</pitfall>
<pitfall>CGEvent.post nécessite la permission Accessibility (AXIsProcessTrusted). Sans elle, aucun événement n'est délivré (échec silencieux). Le prompt AXIsProcessTrustedWithOptions n'ajoute PAS l'app à la liste tout seul de façon fiable — coupler avec deep-link Réglages + polling.</pitfall>
<pitfall>Carbon (IsSecureEventInputEnabled) est « déprécié » sans doc officielle mais reste utilisé partout (iTerm2, Kitty, Chromium, WebKit) et fonctionne sur macOS 14+/Apple Silicon. Ne pas s'inquiéter de la dépréciation; il n'y a pas de remplacement public.</pitfall>
<pitfall>Poser Cmd+V sans délai après avoir écrit le presse-papiers peut coller AVANT que le contenu soit propagé, ou pendant que le focus n'est pas encore rétabli sur l'app cible. VoiceInk attend 0,10 s (prePasteDelay) et 0,01 s entre chaque event.</pitfall>
</pitfalls>
